# Repository Guidelines

## Scope
This project is a generated hackathon workspace managed by OpenClaw. Keep changes local to this folder and treat `hackathon.json`, `OPENCLAW_BRIEF.md`, and `OPENCLAW_ACCEPTANCE.json` as the source of truth.

## Structure
- `app/`: product code
- `tests/`: automated tests
- `artifacts/screenshots/`: UI captures for submission
- `artifacts/demo/`: demo video or supporting files
- `devpost/`: submission copy and structured answers

## Workflow
- Read `hackathon.json` first.
- Update `OPENCLAW_DELIVERABLES.json` as work progresses.
- Do not declare completion until build, tests, and smoke checks pass.
- Prefer small commits with clear messages.

## Verification
Run the project’s install, test, and build commands before publishing. Save proof of verification in `artifacts/verification.md`.
